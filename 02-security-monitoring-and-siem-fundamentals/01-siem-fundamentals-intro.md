
# SIEM Fundamentals - Notes

## What SIEM Is (my understanding)

So SIEM basically stands for Security Information and Event Management. From what I understood while going through this module, it’s basically a system that collects logs from a lot of places (Windows machines, servers, network devices, etc) and then lets analysts look at them in one place. The dashboards part seems pretty important because SOC analysts usually monitor these dashboards to quickly detect weird activity happening in the network.

## My Thought Process

When I started reading this section I was mostly trying to understand:

- where the logs come from
- why we collect them
- how SIEM tools actually help analysts

The key thing I realized is that logs by themselves are useless if you can't query or visualize them properly. SIEM basically organizes them and lets you investigate events faster.

## Key Concepts I Learned

- SIEM collects logs from many sources.
- It allows analysis of security events.
- Dashboards help analysts monitor activity visually.
- Logs can be filtered, searched, and visualized.

Honestly the biggest takeaway here is that **log visibility = security visibility**.
